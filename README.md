# 3D打印耗材RFID数据协议

本仓库定义了一套用于在NFC/RFID标签（如NATG213）上存储3D打印机耗材参数的标准数据格式。该格式为纯文本，易于解析且人类可读，可方便地与Klipper、Moonraker、KlipperScreen等3D打印机固件及用户界面集成。

---

## 协议规范

### 数据格式
- **编码**：UTF-8 纯文本。
- **结构**：
  - 以 `{` 开头
  - 以 `}` 结尾
  - 字段之间用 `;`（分号）分隔
- **语言**：字段内容无限制，建议使用英文以保证广泛兼容性。

### 字段顺序及说明
| 序号 | 字段名（中文） | 字段名（英文） | 数据类型 | 说明 / 示例 |
|------|----------------|----------------|----------|-------------|
| 0    | 头             | Header         | 字符     | 固定为 `{` |
| 1    | 厂商           | Manufacturer   | 字符串   | 例如 `ERYONE` |
| 2    | 耗材主类型     | Material Name  | 字符串   | 例如 `PLA`、`ABS`、`PETG` |
| 3    | 耗材子类型     | Material Supplement | 字符串 | 若无子类型，填一个空格 ` `（半角空格），不可为空 |
| 4    | 颜色           | Color          | 字符串   | 6位十六进制RGB值，例如 `000000` 表示黑色，`FF00FF` 表示紫色 |
| 5    | 透明度         | Transparency   | 字符串   | 2位十六进制，`00` = 完全透明，`FF` = 完全不透明 |
| 6    | 打印最小温度   | Min Temperature | 整数    | 单位：°C，例如 `190` |
| 7    | 打印最大温度   | Max Temperature | 整数    | 单位：°C，例如 `230` |
| 8    | 热床温度       | Bed Temperature | 整数    | 单位：°C，例如 `60` |
| 9    | 直径           | Filament Diameter | 整数  | 存储为整数，实际值需 **除以100** 得到毫米，例如 `175` = 1.75mm |
| 10   | 重量           | Filament Weight | 整数    | 单位：克（g），例如 `1000` |
| 11   | 生产日期       | Production Date | 整数    | 格式：`YYMM`，例如 `2602` 表示 2026年02月 |
| 12   | 尾             | Tail           | 字符     | 固定为 `}` |

### 完整示例
```
{ERYONE;PLA; ;000000;FF;190;230;60;175;1000;2602}
```
**解析含义**：
- 厂商：ERYONE
- 主材料：PLA
- 子类型：无
- 颜色：黑色（#000000）
- 透明度：完全不透明
- 打印温度范围：190°C – 230°C
- 热床温度：60°C
- 直径：1.75mm
- 重量：1000g
- 生产日期：2026年2月

## 物理存储说明（以NATG213为例）
- **标签类型**：NATG213（或其他兼容的NFC Type 2标签，容量不小于180字节）
- **存储器**：45页 × 4字节 = 180字节
- **写入方式**：文本模式，从第0页开始顺序写入整个数据区（覆盖即可）。
- **容量占用**：上述示例字符串约占用50字节，远小于180字节，留有扩展空间。

---

## 使用说明
- 耗材子类型字段**必须至少包含一个字符**；若无子类型，请使用一个半角空格 ` `，而不是空字符串。
- 颜色和透明度均以十六进制字符串存储，**不包含** `#` 前缀。
- 直径存储为整数，代表实际值的**百分之一毫米**（即毫米×100），以避免文本中出现小数点。
- 生产日期格式 `YYMM` 可节省空间，世纪隐含为 2000–2099 年。
